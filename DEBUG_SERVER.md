# MCP Debug Tools

This document explains the debugging and monitoring tools available in the MCP Go implementation.

## Available Debug Tools

### mcp-debug Command

The `mcp-debug` tool provides basic debugging capabilities for MCP servers:

- Server probing and validation
- Protocol compliance checking
- Basic server interaction testing

```bash
# Debug a server running on stdin/stdout
mcp-debug --server ./path/to/server

# Debug with verbose output
mcp-debug --server ./path/to/server --verbose
```

### mcpspy - Traffic Monitoring

Monitor MCP communication in real-time:

```bash
# Monitor communication between client and server
mcpspy -- some-mcp-client
```

### mcpprobe - Server Testing

Test and validate MCP servers:

```bash
# Probe a server for capabilities
mcp-probe --server ./path/to/server

# Test specific functionality
mcp-probe --test-tools --server ./server
```

### mcpdiff - Trace Comparison

Compare MCP traces for testing and debugging:

```bash
# Compare two trace files
mcpdiff trace1.mcp trace2.mcp

# Compare with shadow mode
mcpdiff -compare shadow-trace.mcp
```

### mcp-replay - Session Replay

Record and replay MCP sessions:

```bash
# Record a session
mcp-replay --record session.mcp --server ./server

# Replay a session
mcp-replay --playback session.mcp

# Mock server mode
mcp-replay --mock-server session.mcp
```

## Trace Files

All debugging tools work with `.mcp` trace files that contain JSON-RPC message exchanges. These files are automatically generated by many tools and can be:

- Viewed with `mcpcat`
- Compared with `mcpdiff` 
- Replayed with `mcp-replay`
- Analyzed with `mcpspy`

## Development and Testing

For development, combine these tools:

```bash
# Monitor and record a session
mcpspy -- mcp-replay --record debug.mcp --server ./my-server

# Compare outputs after changes
mcpdiff before.mcp after.mcp

# Test server compliance
mcp-debug --validate --server ./my-server
```
