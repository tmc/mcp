# mcpdiff

A tool for comparing MCP (Model Context Protocol) trace files. It highlights differences between two trace files and supports various comparison modes including semantic JSON comparison, timestamp normalization, and word-level diffs.

## Features

- **Standard Diff Mode**: Compare two separate MCP trace files
- **Compare Mode**: Analyze a single mcptrace file containing shadow server responses
- **Semantic Comparison**: Compare JSON structures ignoring formatting differences
- **Timestamp Handling**: Option to ignore timestamps when comparing
- **ID Normalization**: Option to ignore JSON-RPC IDs
- **Word-Level Diffs**: Show fine-grained differences within JSON content
- **JSON Output**: Export differences in structured JSON format
- **Color-Coded Output**: Visual highlighting of additions and removals

## Usage

### Standard Mode (Two Files)
```bash
# Basic comparison
mcpdiff trace1.mcp trace2.mcp

# Semantic JSON comparison
mcpdiff -s trace1.mcp trace2.mcp

# Ignore timestamps and IDs
mcpdiff -t -i trace1.mcp trace2.mcp

# Word-level diff
mcpdiff --word-diff trace1.mcp trace2.mcp

# JSON output
mcpdiff --json trace1.mcp trace2.mcp > diff.json
```

### Compare Mode (Shadow Server Analysis)
```bash
# Analyze shadow server responses in a single file
mcpdiff -compare shadow-trace.mcp

# Compare mode with semantic comparison
mcpdiff -compare -s shadow-trace.mcp

# Compare mode with JSON output
mcpdiff -compare --json shadow-trace.mcp
```

## Options

- `-t`: Ignore timestamps (default: true)
- `-i`: Ignore IDs when comparing
- `-s`: Compare JSON semantically (ignoring formatting)
- `-o`: Ignore order of messages (best effort matching)
- `-v`: Verbose output (show all lines being considered)
- `-c N`: Number of context lines to show (default: 3)
- `-d`: Only show lines that differ
- `--no-color`: Disable colorized output
- `--json`: Output differences in JSON format
- `--word-diff`: Show word-level differences
- `-compare`: Compare mode for mcptrace files with shadow records

## Compare Mode

Compare mode is designed to work with mcptrace files generated by `mcp-shadow -compare`. These files contain both primary and shadow server responses, allowing you to identify differences between two server implementations from a single recording.

### Expected Format
```
# mcptrace:v1 compare=true
mcp-recv {"jsonrpc":"2.0","method":"calc","id":1} # 1683000000.000 spanid=aaa1
mcp-send {"jsonrpc":"2.0","id":1,"result":42} # 1683000000.100 spanid=aaa2
mcp-send-shadow {"jsonrpc":"2.0","id":1,"result":43} # 1683000000.101 spanid=bbb2 linksto=aaa2
```

### How It Works
1. Parses the mcptrace file, separating primary and shadow records
2. Matches shadow responses to their primary counterparts using span IDs
3. Compares matched pairs and reports differences
4. Outputs in familiar diff format

## Exit Codes

Following standard diff conventions:
- `0`: Files are identical
- `1`: Files differ
- `2`: Error occurred

## Using with Pipes

While direct stdin input isn't supported, you can use temporary files:

```bash
# Create temp files from your pipeline
cat source1.mcp > /tmp/file1.mcp
cat source2.mcp > /tmp/file2.mcp

# Compare the temp files
mcpdiff /tmp/file1.mcp /tmp/file2.mcp
```

## Examples

### Finding Shadow Server Differences
```bash
# Record shadow traffic with mcp-shadow
mcp-shadow -primary "server-v1" -shadow "server-v2" -compare -o shadow.mcp

# Analyze differences
mcpdiff -compare shadow.mcp
```

### Advanced Comparison
```bash
# Semantic comparison ignoring timestamps
mcpdiff -compare -s -t shadow.mcp

# Export differences to JSON for further analysis
mcpdiff -compare --json shadow.mcp | jq '.diffs[] | select(.removed[0].result != .added[0].result)'
```
