# MCP Exec Server Specification

## Overview

The `mcp-exec` server is a Model Context Protocol (MCP) server that provides a tool for executing shell commands on the server-side. This capability allows MCP clients to perform various tasks, such as running scripts, interacting with the file system, and executing system utilities.

**WARNING**: This server provides the ability to execute arbitrary commands on the host system. It should be used with extreme caution and only in trusted environments with appropriate security measures in place.

## Tool: `exec`

### Description

Executes a shell command on the server and returns the output.

### Input Parameters

| Parameter           | Type     | Required | Default | Description                                                                                                                                                                |
| :------------------ | :------- | :------- | :------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `command`           | `string` | Yes      |         | The shell command to execute.                                                                                                                                              |
| `arguments`         | `array`  | No       | `[]`    | An array of string arguments to pass to the command.                                                                                                                      |
| `environment`       | `object` | No       | `{}`    | A set of environment variables to set for the command's execution environment. The object's keys are the variable names, and the values are the variable values. |
| `working_directory` | `string` | No       |         | The working directory in which to execute the command.                                                                                                                    |
| `timeout`           | `number` | No       | `60`    | The maximum execution time for the command in seconds.                                                                                                                   |

### Output

| Field       | Type     | Description                                      |
| :---------- | :------- | :----------------------------------------------- |
| `stdout`    | `string` | The standard output of the command.             |
| `stderr`    | `string` | The standard error of the command.             |
| `exit_code` | `number` | The exit code of the command.                   |

### Errors

| Code              | Description                                                   |
| :---------------- | :------------------------------------------------------------ |
| `INVALID_PARAMS`  | Returned if the input parameters are invalid or missing.    |
| `EXECUTION_FAILED`| Returned if the command execution fails.                    |
| `TIMEOUT`         | Returned if the command execution exceeds the specified timeout. |
| `NOT_ALLOWED`     | Returned if the command is not allowed due to security restrictions. |

## Security Considerations

The `exec` tool provides arbitrary command execution capabilities, which poses significant security risks if not handled carefully. Implementations must adhere to the following security guidelines:

1. **User Consent:** Implementations must obtain explicit user consent before executing any command. The user should be presented with the full command and its arguments and have the opportunity to approve or deny the execution.
2. **Command Whitelisting:** Implementations should provide a mechanism to restrict the set of allowed commands. This can be achieved through a whitelist of permitted commands or by using a restricted shell.
3. **Input Validation:** All input parameters must be carefully validated and sanitized to prevent command injection vulnerabilities.
4. **Sandboxing:** Consider running the commands in an isolated environment, such as a container or virtual machine, to limit the impact of potential security breaches.
5. **Resource Limits:** Enforce resource limits (CPU, memory, network) on the executed commands to prevent denial-of-service attacks.
6. **Least Privilege:** Run the server with the least privilege necessary to perform its tasks. Avoid running as root or with administrative privileges.

## Configuration

### Example: Claude Desktop

To use the `mcp-exec` server with Claude Desktop, add the following configuration to your `claude_desktop_config.json` file:

```json
{
  "mcpServers": {
    "exec": {
      "command": "docker",
      "args": ["run", "-i", "--rm", "mcp/exec"]
    }
  }
}
```

This configuration tells Claude Desktop to start the `mcp-exec` server using Docker. You can replace `mcp/exec` with the appropriate image name or command if you are not using Docker.

## Building

### Docker

To build the Docker image for the `mcp-exec` server, run the following command from the root of the MCP project:

```bash
docker build -t mcp/exec -f src/exec/Dockerfile .
```
