# Example: Per-Line Coverage Analysis Output

## Original Test File (testdata/coverage_test.txt)
```
# Test multiple MCP tools
exec mcpdiff file1.mcp file2.mcp
stdout 'Files match'

exec mcpcat -color=never file1.mcp
stderr 'initialize'

exec mcpspy -- echo "test message"
stdout 'test message'

exec mcpdiff file1.mcp file3.mcp
! stdout 'Files match'

-- file1.mcp --
mcp-send {"jsonrpc":"2.0","method":"initialize","id":1}

-- file2.mcp --
mcp-send {"jsonrpc":"2.0","method":"initialize","id":1}

-- file3.mcp --
mcp-send {"jsonrpc":"2.0","method":"shutdown","id":1}
```

## Annotated Output with Per-Line Coverage

## Benefits of Per-Line Analysis

1. **Identify redundant tests**: Line 11 adds only 0.6% coverage
2. **Find coverage gaps**: No tests for mcpcat error handling
3. **Optimize test execution**: Focus on high-value lines
4. **Visual feedback**: Heat maps show coverage at a glance
5. **Incremental analysis**: See what each line adds

This approach would revolutionize test optimization by showing exactly which test lines provide the most value for coverage.
